GET / HEAD

/getPokemon

Query Parameters

  • name (optional) - The string of characters to be found in the Pokemon

  • id (optional) - The ID number for a specific Pokemon

If no parameters are inputted, Bulbasaur's information is returned instead.

Return

Returns an array of Pokemon JSON objects

Format: { "response": [ { "id": number, "num": string, "name": string, "img": string, "type": [ string ], "height": string, "weight": string, "weaknesses": [ string ], "next_evolution": [ { "num": string, "name": string } ] } ] }

Example

/getPokemon?name=pikachu

{ "response": [ { "id": 25, "num": "025", "name": "Pikachu", "img": "http://www.serebii.net/pokemongo/pokemon/025.png", "type": [ "Electric" ], "height": "0.41 m", "weight": "6.0 kg", "weaknesses": [ "Ground" ], "next_evolution": [ { "num": "026", "name": "Raichu" } ] } ] }

/getPokemonType

Query Parameters

  • typeA (optional) - The string representing a Pokemon's first type

  • typeB (optional) - The string representing a Pokemon's second type

If no parameters are inputted, all Normal Type Pokemon are returned instead.

Return

Returns an array of Pokemon JSON objects

Format: { "response": [ { "id": number, "num": string, "name": string, "img": string, "type": [ string ], "height": string, "weight": string, "weaknesses": [ string ], "next_evolution": [ { "num": string, "name": string } ] } ] }

Example

/getPokemonType?typeA=Dragon&typeB=

/getRandomPokemon

Query Parameters

No query parameters necessary

Return

Returns a Pokemon JSON object

Format: { "response": { "id": number, "num": string, "name": string, "img": string, "type": [ string ], "height": string, "weight": string, "weaknesses": [ string ], "next_evolution": [ { "num": string, "name": string } ] } }

Example

/getRandomPokemon

{"response":{"id":79,"num":"079","name":"Slowpoke","img":"http://www.serebii.net/pokemongo/pokemon/079.png","type":["Water","Psychic"],"height":"1.19 m","weight":"36.0 kg","weaknesses":["Electric","Grass","Bug","Ghost","Dark"],"next_evolution":[{"num":"080","name":"Slowbro"}]}}

POST

/addPokemon

Body Params

  • name - The string of characters to be found in the Pokemon

  • typeA - The string representing a Pokemon's first type

  • typeB (optional) - The string representing a Pokemon's second type

Return

Returns a message confirming the Pokemon's creation

Format: { "message": "dog created successfully!" }

Example

/addPokemon

Body: {name: 'Monkey', 'typeA': 'Normal', 'typeB': ''}

{ "message": "Monkey created successfully!" }

/addEvolution

Body Params

  • preEvolution - The string representing an exisiting Pokemon

  • evolution - The string representing another exisiting Pokemon

Return

Does not return a response

Example

/addPokemon

Body: {preEvolution: 'Monkey', 'evolution': 'Mankey'}

This would end up linking Monkey to Mankey in its next_evolution property.
